1. PROJECT PURPOSE
============================================================

IL Quantum is a browser-based single-page enterprise dashboard portal designed as a unified data hub for intelligent insurance / health / corporate reporting decisions.

The portal allows users to:

1. Enter through a branded animated homepage.
2. Select role-based access through navigation cards.
3. Access a dashboard page for data upload and analysis.
4. Upload CSV / Excel files or folders.
5. Parse and combine uploaded datasets.
6. Dynamically generate filters.
7. Render dashboard sections as tables and charts.
8. Toggle all sections between table and graph view.
9. Create custom charts.
10. Export raw data, graph data, table data, PPT, and PDF.
11. Customize theme, color, dark mode, font size, Top N records, chart colors, parser delay, file size, and file count.
12. Use compact uploaded file progress popup during parsing.
13. Use animated dashboard menu icons and chart action icons.

============================================================

SUPPORTED FILE TYPES:

.xls

UPLOAD OPTIONS:

2. Multi-file upload through Files button.

multiple

IMPORTANT GLOBAL VARIABLES:
uploadedFiles = []
fileQueue = []
isProcessing = false
currentFileMeta = null
combinedData = []
filteredData = []
uniqueColumns = []

UPLOAD FLOW:
1. User selects files or folder.
2. handleFileUpload(files) runs.
3. File panel opens automatically.
4. File panel opens in compact mode.
5. Each file is validated.
6. Valid file metadata is created.
7. File is added to uploadedFiles.
8. File is added to fileQueue.
9. processFiles() starts.
10. CSV or Excel parser runs.
11. Data is normalized.
12. Rows are appended to combinedData.
13. Dashboard is rendered.

VALIDATIONS:
1. File type must be csv, xlsx, or xls.
2. File size must be within CONFIG.MAX_FILE_SIZE_MB.
3. Total uploaded files must not exceed CONFIG.MAX_FILE_COUNT.

============================================================
12. FILE PARSER FUNCTIONS
============================================================

handleFileUpload(files)
Purpose:
Handles selected files/folders.

Steps:
1. Checks if files exist.
2. Opens file popup.
3. Adds compact class to file popup.
4. Removes collapsed class.
5. Validates file count.
6. Validates file extension.
7. Validates file size.
8. Creates file metadata.
9. Adds metadata to uploadedFiles.
10. Adds file to fileQueue.
11. Calls updateFilePanel().
12. Starts processFiles() if not already processing.

processFiles()
Purpose:
Processes files one by one from fileQueue.

Steps:
1. If queue is empty:
   - Stop processing.
   - Reset current file.
   - Update progress.
   - Update file panel.
   - Render dashboard.
   - Auto-close popup if no issues.
   - Show final toast.
2. If queue has file:
   - Pick next file.
   - Set currentFileMeta.
   - Mark as Processing.
   - Update progress.
   - Parse CSV or Excel.

parseCSV(file, fileMeta)
Purpose:
Reads and parses CSV file.

Uses:
PapaParse if available.
Fallback parser if PapaParse is unavailable.

Steps:
1. Reads CSV using FileReader.
2. Parses rows.
3. Handles empty file.
4. Normalizes rows.
5. Appends data.
6. Marks file uploaded.
7. Runs stable progress processing.
8. Continues next file.

parseExcel(file, fileMeta)
Purpose:
Reads and parses Excel files.

Uses:
XLSX library.

Steps:
1. Reads workbook.
2. Loops through all sheets.
3. Converts each sheet to JSON.
4. Adds __sheetName.
5. Handles empty file.
6. Normalizes rows.
7. Appends data.
8. Marks file uploaded.
9. Runs stable progress processing.
10. Continues next file.

runStableProcessingProgress(rows, callback)
Purpose:
Runs chunk-based progress simulation from 70% to 100%.

Uses:
CONFIG.TABLE_RENDER_CHUNK
CONFIG.TABLE_RENDER_DELAY

updateProgress(p)
Purpose:
Updates hidden parser progress and current file progress.

fallbackCSVParse(text)
Purpose:
Basic CSV parser if PapaParse is unavailable.

normalizeRows(rows, fileMeta)
Purpose:
Normalizes all column names and adds file metadata.

Added metadata columns:
__fileId
__fileName
__fileType
__fileSize
__monthYear

appendData(rows)
Purpose:
Adds normalized rows to combinedData and rebuilds uniqueColumns.

validateRequiredColumns()
Purpose:
Checks missing required columns if CONFIG.REQUIRED_COLUMNS is configured.
